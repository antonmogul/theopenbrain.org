01 · Product vision

Open Brain is an openly-licensed textbook on visual neuroscience that treats the figure, not the paragraph, as the unit of explanation. Every chapter pairs editorial-quality prose with a fullscreen interactive diagram that the prose drives via scroll — the diagram is the protagonist, the writing is its narration.

The product is free, syncs the reader's progress and notes across devices, and ships in light and dark modes that follow OS preference by default.

Audience

  • Primary — undergraduates and self-directed learners working through introductory or intermediate neuroscience content.
  • Secondary — educators who assign chapters and want their students to engage with diagrams that aren't passive PDFs.
  • Tertiary — researchers, clinicians, and curious adults who want a high-quality refresher on a specific topic.

Core jobs-to-be-done

  1. Read a chapter in a single continuous session — figures, prose, and recall checks integrated.
  2. Resume where I left off — across devices, across sessions, with my notes intact.
  3. Lock in what I just read — quizzes and flashcards immediately after the section.
  4. Manipulate a phenomenon to understand it — not watch a video of someone else doing it.
  5. Make it mine — highlights, notes, type/theme preferences that persist.

02 · Design principles

The figure is the protagonist

On desktop, the interactive diagram occupies a sticky 60% pane while the prose scrolls past on the right and updates which figure is in view. Engineers must preserve this hierarchy: the diagram is never collapsed into a "show figure" footnote.

Editorial, not corporate

Open Brain looks like a thoughtful indie magazine that happens to be a software product — heavy serif display type, hairline rules, generous whitespace, mono-spaced metadata. Avoid: gradient-heavy heroes, drop shadows, rounded corners over 6px on structural surfaces, emoji, "AI"-flavored visual language.

Quiet defaults, loud accents

The page is paper-warm off-white (light) or near-black (dark). One accent — magenta — does the heavy lifting for active states, progress, and inline emphasis. The teal is reserved for "completed" and key-takeaway callouts. Highlighter swatches (yellow / pink / blue / green / purple) are user-applied only.

Calm interactions

Diagram steps cross-fade over 320ms. Theme transitions ease over 220ms. No bouncy springs, no flashing badges, no "you've got mail!" enthusiasm. Animation exists to clarify state changes, not to entertain.

Mobile is its own product, not a squeeze

On mobile, the side-by-side scroll-coupled layout collapses into a single integrated stream — diagrams render inline, prose flows around them, and an expand-to-fullscreen control lets the reader manipulate a figure without losing place in the prose.

Theme behavior

System preference is the default. On first load, read prefers-color-scheme and set data-theme on <html>. The user can override to "Light" or "Dark" in Profile → Theme & Accents; their choice persists to localStorage.ob.theme.

When the user is in system mode and the OS preference changes mid-session, the app must respond live via a matchMedia change listener — already wired in useThemeManager.

07 · Interactive diagrams

Each chapter has 1–N interactive figures rendered as React components. The Reader's sticky pane swaps between them based on scroll. Three reference implementations ship in the prototype:

ComponentFileDemonstrates
RefractionDiagramcomponents/refraction-diagram.jsxToggle myopia / hyperopia / astigmatism. Shows ray paths, focal point relative to retina, corrective lens overlay.
PhotoTransductioncomponents/photo-transduction.jsx8-step cascade. Each step has a prose caption, an SVG state, and animates in 320ms cross-fade. Auto-advance toggle.
ConeExplorercomponents/diagrams.jsxThree overlapping cone-sensitivity curves (S/M/L). Slider controls test wavelength; readout shows excitation ratio.

Diagram chassis contract

  • Component must fill its container (position:relative; width/height: 100%) — no fixed dimensions.
  • State that should persist between scroll-out and scroll-back lives in localStorage under ob.diagram.<id>.<key>.
  • Honor prefers-reduced-motion via theme.reduceMotion setting — disable cross-fades and parallax.
  • Expose at minimum: a fullscreen-expand button (mobile), a reset button, and screen-reader text describing the current state.

Step diagrams (cascade-style)

Render as a vertical sequence of steps inside the figure pane. Active step is highlighted; completed steps are dimmed; future steps show only their number. "Auto-play step diagrams" preference (Profile → Reading) controls whether scroll-into-view auto-advances them.

08 · Mobile

Mobile branches via a viewport check (useIsMobile @ <768px) and renders MobileApp from components/mobile.jsx. The Reader becomes a single integrated stream:

  1. Slim sticky app bar (menu + chapter title + progress dot).
  2. Hero block.
  3. Prose sections interleaved with inline-rendered diagrams (each with a fullscreen-expand control).
  4. Inline quiz/flashcard checkpoints between sections.
  5. Two-state bottom sheet (peek / expanded) for Notebook & Tools — replaces the desktop floating panel.

Profile on mobile uses a single-column variant (ProfileMobileScreen) — top bar, avatar+name header, theme cards 3-up, segmented size control, accent dots, email-prefs list, sign-out button at the foot.

11 · Tech stack recommendation

  • Frontend — React 18 + TypeScript + Vite. Routing: TanStack Router or React Router 6 (the prototype's hash router is for demo only).
  • Styling — CSS custom properties from brand.css consumed via vanilla CSS modules or Vanilla Extract. Do not introduce Tailwind, styled-components, or emotion — the design system is token-first and the prototype intentionally uses inline styles + utility classes that map 1:1 to tokens.
  • State — Zustand or Jotai for app state; TanStack Query for server data. No Redux.
  • Backend — opinion-free; FastAPI / Rails / Hono all fine. PostgreSQL for relational data, plus a simple object store for chapter assets.
  • Auth — WorkOS or Clerk for SSO; magic-link via Postmark for password-light flows.
  • Content — chapters authored in MDX, with custom MDX components mapping to the diagram React components. Each chapter is a directory: chapters/02-vision/index.mdx + co-located figures.
  • AI Chat — the prototype uses window.claude.complete. Production should proxy through a backend endpoint that injects the current chapter context and rate-limits per user.

12 · Accessibility

  • Contrast — All text passes WCAG AA in both themes. The mute color (#6b6b66 on bg #f7f5f0) is intentionally close to the limit at 14px+ — never use it under 13px.
  • Focus — 2px solid magenta outline with 2px offset, never outline:none.
  • Keyboard — Reader: ↑/↓ scroll prose; ←/→ change pinned figure; K opens notebook; ? opens shortcuts. Quiz: 1–4 select answer, Enter advances.
  • Reduced motion — Respect prefers-reduced-motion: reduce by default; user can also toggle it on explicitly. Disables cross-fades, scroll-coupled diagram swaps (figure pane just changes instantly), and modal scale-ins.
  • Screen readers — Each diagram exposes a live-region textual description of its current state. Section navigation announces section number + title.
  • Min hit target — 44×44 on mobile, 30×30 on desktop with at least 4px padding around clickable text.
